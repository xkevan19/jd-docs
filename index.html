Introduction

Documentation v1.2

The Jesiah Desaine Media Dashboard is a centralized, web-based control panel designed for administrators to manage digital media assets and website content. It provides a comprehensive suite of tools to efficiently upload, organize, categorize, and optimize images and other media, ensuring a streamlined workflow and optimal website performance.

Latest Release

1.1 Overview

Core

The primary purpose of this dashboard is to empower administrators with complete control over the media content displayed on the Jesiah Desaine Media website. Key features include:

Media Upload

Seamlessly upload images in various formats (JPG, PNG, GIF, etc.).

Categorization

Organize media into predefined categories (Client, Restaurant, Events) for easy retrieval.

Binary Blade

Convert and resize images to WebP format, optimizing for web performance and reducing storage.

Media Gallery

Browse, search, and manage all uploaded media assets.

Dashboard Overview

View at-a-glance statistics on media usage, storage consumption, and recent activity.

User Permissions

Manage access levels and editing rights for different team members.

This documentation is intended for administrators with approved access to the Media Dashboard.

Architecture

Documentation v1.2

The Jesiah Desaine Media Dashboard is built on a modern, serverless architecture leveraging the power of Supabase for backend services, Netlify Functions for server-side logic, and a responsive front-end built with Tailwind CSS. This architecture was chosen for its scalability, security, and ease of development and maintenance.

3.2 Front-End

The front-end of the Jesiah Desaine Media Dashboard is built with a focus on responsiveness, real-time updates, and a user-friendly experience. It utilizes modern web technologies and a component-based structure for maintainability and scalability.

3.3 Back-End

The back-end of the Jesiah Desaine Media Dashboard is powered by Supabase, a comprehensive platform providing authentication, database, and storage services. This serverless architecture enables scalability, security, and real-time capabilities.

Supabase

Authentication

User authentication is managed by Supabase Auth, utilizing email/password login.

Database (PostgreSQL)

The application uses Supabase's PostgreSQL database to store structured data. The database schema is designed to efficiently manage users, uploads, categories, and activity logs.

Database Schema
Table Name Column Name Data Type Description
profiles id UUID (Primary Key) Unique user ID (from Supabase Auth)
profiles full_name TEXT User's full name
uploads id SERIAL (Primary Key) Unique upload ID
uploads image_url TEXT URL of the image in Supabase Storage
uploads category TEXT Category the image belongs to
uploads file_size INTEGER File size in bytes
uploads file_type VARCHAR MIME type of file
uploads created_at TIMESTAMPTZ Timestamp of upload
categories id SERIAL (Primary Key) Unique category ID
categories name TEXT Name of the category
activity_logs id SERIAL (Primary Key) Unique activity log ID
activity_logs user_id UUID ID of the user who performed the action
activity_logs user_name TEXT User's Name
activity_logs action TEXT Description of the action performed (e.g., "Upload")
activity_logs item_name TEXT Name of the affected item (e.g., filename)
activity_logs category TEXT Category involved (if applicable)
activity_logs created_at TIMESTAMPTZ Timestamp of the activity
Relationships

The uploads table has a many-to-one relationship with the categories table via the category column. This means that each upload belongs to one category, but a category can have many uploads.

3.4 API Integrations

The Jesiah Desaine Media Dashboard integrates with several external APIs to provide enhanced functionality and content. These integrations are managed securely and efficiently, leveraging serverless functions for communication.

External API Integrations

OpenWeatherMap API

Provides current weather data for display on the dashboard. The API key is stored securely as a Netlify environment variable.

JokeAPI

Provides random jokes for display on the dashboard. This API does not require an API key.

CDN Services

The following Content Delivery Networks (CDNs) are used to efficiently deliver static assets, improving performance and reducing latency:

  • Tailwind CSS CDN: Delivers the Tailwind CSS framework.
  • Supabase CDN: Delivers Supabase client libraries.
  • JSZip CDN: Delivers the JSZip library.
  • Chart.js CDN: Delivers the Chart.js library.

3.5 Security Considerations

Security is a paramount concern in the design and development of the Jesiah Desaine Media Dashboard. Multiple layers of security measures are implemented to protect user data and ensure the integrity of the application.

Authentication

User authentication is handled securely by Supabase Auth, using email/password login. Supabase Auth provides built-in security features, including protection against common web vulnerabilities.

Authorization

Access to specific features is controlled through application logic.

Data Validation

Input validation is performed to ensure data integrity and prevent security vulnerabilities. This includes:

  • File type and size limits are enforced for uploads.

Environment Variables

Sensitive information, such as Supabase keys and API keys, are stored securely using Netlify environment variables. This prevents these secrets from being exposed in the codebase.

HTTPS

The application is served exclusively over HTTPS, ensuring all communication between the client and server is encrypted.

Regular Updates

Dependencies, as listed in package.json, are regularly reviewed and updated to apply security patches and mitigate potential vulnerabilities.

3.6 Deployment

The Jesiah Desaine Media Dashboard is deployed on Netlify, a platform providing hosting, continuous deployment, and serverless functions. This setup ensures high availability, scalability, and ease of maintenance.

Netlify Deployment

The application is hosted on Netlify, leveraging its global CDN for fast content delivery.

Continuous Deployment

Continuous Deployment is enabled, with automatic deployments triggered by pushes to the main branch of the GitHub repository. This ensures that the latest version of the application is always live.

Environment Variables

Environment variables, including sensitive keys and configuration settings, are managed securely within the Netlify UI. This ensures that secrets are not exposed in the codebase.
